# 모임 설정 가이드

Want2Eat은 모임(연인) 기반으로 동작합니다. 같은 모임에 속한 구성원들은 모든 장소를 공유하고 코멘트를 작성할 수 있습니다.

## 초기 모임 생성

현재는 하나의 고정된 모임을 사용합니다. Supabase 대시보드에서 다음 SQL을 실행하여 모임을 생성하세요.

### 1. 모임 생성

```sql
-- 모임 생성 (이름은 자유롭게 변경 가능)
INSERT INTO groups (id, name) 
VALUES ('00000000-0000-0000-0000-000000000001', '연인') 
ON CONFLICT DO NOTHING;
```

### 2. 모임 구성원 추가

환경 변수에서 설정한 Supabase User ID를 사용하여 구성원을 추가합니다.

```sql
-- 기정, 하늘을 모임에 추가
-- USER1_SUPABASE_ID와 USER2_SUPABASE_ID를 실제 값으로 변경하세요
INSERT INTO group_members (group_id, user_id) 
VALUES
  ('00000000-0000-0000-0000-000000000001', 'USER1_SUPABASE_ID'),
  ('00000000-0000-0000-0000-000000000001', 'USER2_SUPABASE_ID')
ON CONFLICT DO NOTHING;
```

### 3. 환경 변수 설정

`.env.local` 파일에 모임 ID를 추가하세요:

```env
NEXT_PUBLIC_GROUP_ID=00000000-0000-0000-0000-000000000001
```

또는 Vercel 환경 변수에 추가하세요.

## 모임 동작 방식

1. **장소 공유**: 같은 모임에 속한 모든 사용자는 모든 장소를 볼 수 있습니다.
2. **장소 저장**: 장소를 저장하면 자동으로 모임에 연결됩니다.
3. **코멘트**: 모임 구성원들은 모든 장소에 코멘트를 작성할 수 있습니다.
4. **포스팅**: 모임 구성원들은 모든 장소에 방문 기록을 작성할 수 있습니다.

## 주의사항

- 현재는 하나의 모임만 지원합니다.
- 모임 ID는 환경 변수 `NEXT_PUBLIC_GROUP_ID`에서 가져옵니다.
- 모임에 속하지 않은 사용자는 장소를 볼 수 없습니다.
